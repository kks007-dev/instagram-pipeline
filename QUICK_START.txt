QUICK 5-STEP SETUP (see SETUP.md for detailed guide):

1. TELEGRAM BOT
   → Chat with @BotFather
   → /newbot → get BOT_TOKEN

2. ASSEMBLYAI
   → Sign up at assemblyai.com
   → Get API_KEY from dashboard

3. CLAUDE CODE ROUTINE
   → Visit claude.ai/code/routines
   → Create "Instagram Video Analyzer"
   → Paste routine-prompt.md as system prompt
   → Add Google Tasks connector (OAuth)
   → Enable API trigger → copy ENDPOINT & TOKEN

4. DEPLOY WEBHOOK SERVER
   Railway:  railway.app → New Project → Connect GitHub
   Render:   render.com → New Web Service → Connect GitHub
   Local:    npm install && npm start (use ngrok for tunnel)

5. REGISTER TELEGRAM WEBHOOK
   curl -X POST https://api.telegram.org/bot{TOKEN}/setWebhook \
     -H "Content-Type: application/json" \
     -d '{"url": "https://your-deployed-url.com/webhook"}'

ENVIRONMENT VARIABLES (copy .env.example → .env):
- TELEGRAM_BOT_TOKEN         (from BotFather)
- ROUTINE_API_ENDPOINT       (from claude.ai/code/routines)
- ROUTINE_BEARER_TOKEN       (from Routine API trigger settings)
- ASSEMBLYAI_API_KEY         (in Routine settings, not webhook server)
- PORT                       (default: 3000)
- NODE_ENV                   (production/development)

TESTING:
1. Use example-test-payload.json with test-routine-api.sh
2. Set environment variables: ROUTINE_API_ENDPOINT, ROUTINE_BEARER_TOKEN
3. Run: ./test-routine-api.sh example-test-payload.json
4. Check Google Tasks for new task within 2-5 minutes

COMMON ISSUES & FIXES:

"Webhook not receiving messages"
→ Check: curl https://your-url/health
→ Verify: curl https://api.telegram.org/bot{TOKEN}/getWebhookInfo

"Routine not triggering"
→ Check ROUTINE_API_ENDPOINT & ROUTINE_BEARER_TOKEN are correct
→ Test: ./test-routine-api.sh example-test-payload.json
→ Review Claude Code Routine logs

"No Google Tasks created"
→ Verify Google Tasks connector is authenticated in Routine settings
→ Check that Google account is accessible
→ Review Routine logs for Google Tasks API errors

"Transcription fails"
→ Verify AssemblyAI API key is set in Routine environment
→ Check remaining API quota (free: 600 min/month)
→ Ensure video file isn't corrupted
